# Diabetes Risk Prediction

An end-to-end project that predicts diabetes risk from clinical measurements. Covers the full lifecycle: exploratory data analysis, model training, REST API serving, containerisation, experiment tracking, and automated CI/CD with weekly retraining.

---

## Overview

| | |
|---|---|
| **Dataset** | Public dataset (768 patients, 8 features) |
| **Algorithm** | XGBoost binary classifier |
| **API** | FastAPI В· single + batch prediction endpoints |
| **Tracking** | MLflow experiment tracking + model versioning |
| **Deployment** | Docker + docker-compose |
| **CI/CD** | GitHub Actions В· lint, test, build, push, retrain |

## Exploratory data analysis

Four analysis scripts answer progressively deeper questions about the data.

| Script | Question answered | Key outputs |
|---|---|---|
| `01_descriptive_analysis.py` | What does the data look like? | Summary stats, missing value audit, correlation heatmap |
| `02_diagnostic_analysis.py` | Why does diabetes occur in this data? | t-tests, Mann-Whitney U, Cohen's d, odds ratios |
| `03_predictive_analysis.py` | Who will develop diabetes? | 6-model comparison, XGBoost tuning, SHAP feature importance |
| `04_prescriptive_analysis.py` | What should we do about it? | Risk stratification, intervention priority matrix, per-patient recommendations |

Run any script directly:

```bash
python 01_descriptive_analysis.py
```

Outputs are saved to the same directory as `.png` plots and `.txt` reports.

## Quickstart

### 1. Train the model

```bash
python train.py
```

This loads `diabetes.csv`, preprocesses (KNN imputation + RobustScaler), trains XGBoost, evaluates on a held-out 20% test set, logs everything to MLflow, and saves four files to `model_artifacts/`.

### 2. Start the API

```bash
uvicorn app.main:app --reload --port 8000
```

The API is now live at `http://localhost:8000`. Interactive docs at `http://localhost:8000/docs`.

### 3. Open the UI

Double-click `ui.html` in your file explorer. Use the sliders to set patient values and click **Run prediction**.

## Docker

### Run with docker-compose (recommended)

Starts both the prediction API and the MLflow UI in one command.

```bash

docker-compose up --build -d

# View logs
docker-compose logs -f api

# Stop

```

| Service | URL |
|---|---|

| MLflow UI | http://localhost:5000 |

## MLflow experiment tracking

Every `train.py` run is logged automatically.

```bash
# Start the MLflow UI
mlflow ui --backend-store-uri sqlite:///mlflow_tracking/mlflow.db
# Open http://localhost:5000
```

Each run records:

- Hyperparameters (n\_estimators, max\_depth, learning\_rate, subsample, colsample\_bytree)
- Metrics (ROC-AUC, average precision, F1, accuracy, CV mean/std)
- Artifacts (model JSON, imputer, scaler, classification report)
- Tags (model type, dataset, production/staging status)

## Model performance

Trained on 80% of the dataset (614 samples), evaluated on the held-out 20% (154 samples).

| Metric | Value |
|---|---|
| ROC-AUC | 0.8148 |
| Average precision | 0.647 |
| Accuracy | 73.4% |
| F1 (diabetic class) | 0.661 |
| 5-fold CV AUC | 0.8339 В± 0.019 |

**Classification report**

```
              precision    recall  f1-score   support
 No Diabetes       0.84      0.73      0.78       100
    Diabetes       0.60      0.74      0.66        54
    accuracy                           0.73       154
```

**Top features by SHAP importance:** Glucose В· BMI В· Age В· Pregnancies В· DiabetesPedigreeFunction

**Preprocessing pipeline**

1. Replace biologically impossible zeros with `NaN` (Glucose, BloodPressure, SkinThickness, Insulin, BMI)
2. `KNNImputer(n_neighbors=5)` вЂ” fills missing values using nearest neighbours
3. `RobustScaler` вЂ” normalises using median/IQR, robust to outliers
4. `XGBClassifier` with `scale_pos_weight` to handle the 65/35 class imbalance
